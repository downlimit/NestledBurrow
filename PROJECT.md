# NestledBurrow: текущий вектор

## Где находится проект

Создан минимальный браузерный проект на Phaser 4.2.1 и Vite. Он собирается и показывает стартовую сцену. Репозиторий `downlimit/NestledBurrow` подключён; начальный проект слит в `main`. Публикация через GitHub Pages подготовлена в текущем PR, но публичной игровой ссылки пока нет.

## Текущая цель

Наладить минимальный рабочий путь:

`задача в ChatGPT → изменение кода → проверка → GitHub → запуск игры по ссылке`

## Следующий шаг

1. Проверить и слить PR с настройкой GitHub Pages.
2. В настройках репозитория включить GitHub Pages через источник GitHub Actions, если это ещё не сделано.
3. Дождаться успешного workflow `Deploy GitHub Pages` и проверить публичную ссылку.
4. После этого сделать первую техническую игровую проверку: пустая комната и временный объект игрока с движением.

## Текущий блокер

Публикация ещё не проверена на GitHub Pages: нужно слить PR с workflow, включить Pages через GitHub Actions и дождаться успешного деплоя.

## Действующие решения

- Основное общение и постановка задач происходят в ChatGPT с телефона или компьютера.
- Игра создаётся как браузерная 2D-игра на Phaser 4.
- Код и проектные документы должны храниться в GitHub.
- Основной чат управляет направлением работы; кодовый агент выполняет ограниченные задачи, после чего результат проверяется.
- Проектные знания разделяются на тематические «здания» только по фактической необходимости; список зданий заранее не придумывается.
