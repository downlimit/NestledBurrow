# NestledBurrow: текущий вектор

## Где находится проект

Создан минимальный браузерный прототип на Phaser 4.2.1 и Vite. Он показывает единый непрерывный top-down мир на CC0-тайлах Basic Village, маленького анимированного персонажа Kenney, движение с клавиатуры и мобильный виртуальный джойстик. В правом верхнем углу отображается идентификатор опубликованной сборки.

Репозиторий: `downlimit/NestledBurrow`.

Опубликованная игра: https://downlimit.github.io/NestledBurrow/.

## Текущая цель

Развивать игровой прототип короткими проверяемыми итерациями, не добавляя инфраструктурные усложнения без фактической необходимости.

## Следующий шаг

## Текущий блокер

Текущих инфраструктурных блокеров нет.

## Рабочий цикл

`ChatGPT формулирует ограниченную задачу → Codex работает в отдельной ветке без открытого PR → выполняет обязательный self-test из AGENTS.md → PR создаётся только после успешных локальных проверок → ChatGPT проверяет код, CI и фактический визуальный результат → при необходимости исправляет ветку → выполняет merge → ждёт pages/live: success → сообщает «можно запускать»`

## Действующие решения

- Основное общение, проектирование и постановка задач происходят в обычном ChatGPT-чате.
- Лимиты Codex расходуются преимущественно на непосредственную работу с кодом.
- Игра создаётся как браузерная 2D-игра на Phaser 4.
- Код и проектные документы хранятся в GitHub.
- Изменения проходят через отдельные ветки и pull request.
- Черновой PR не открывается во время разработки: он создаётся один раз после прохождения локальных проверок, чтобы не запускать промежуточные CI и не создавать лишние уведомления.
- Корневой `AGENTS.md` задаёт обязательный порядок self-test для Codex: автоматические проверки, запуск игры и проверка фактических изменённых состояний.
- **Basic Village Tileset от Forchild является приоритетным и каноническим набором окружения.** Другие наборы используются только для отсутствующих категорий или после отдельного визуального согласования.
- Персонаж пока использует совместимые CC0-ассеты Kenney.
- Сторонний spritesheet нельзя подключать по угаданным числовым индексам. Сначала проверяются геометрия листа, подписанные кадры и примеры автора; выбранные индексы централизуются в конфигурации и подтверждаются фактическим preview.
- Визуальное изменение нельзя сливать только на основании успешной сборки и проверки числовых конфигураций: до merge нужно осмотреть фактический рендер и workflow-артефакт предпросмотра.
- Pull request должен пройти workflow `Pull Request Check`, который запускает проверки ввода, ассетов, layout, preview и production-сборку.
- После merge workflow публикации создаёт `/version.json` с полным и коротким SHA сборки.
- Workflow публикации самостоятельно проверяет живой `/version.json` и выставляет статус `pages/live`.
- Изменение считается опубликованным только после `pages/live: success`.
- Формулировка «можно запускать» используется только после успешной проверки живой версии.
- Для обхода браузерного кэша ссылка на игру может дополняться параметром `?v=<короткий SHA>`.
- Реализованы клавиатурное управление и мобильный виртуальный джойстик с обработкой отмены и потери активного касания.
- Проектные знания разделяются на тематические «здания» только по фактической необходимости; список зданий заранее не придумывается.

## Текущая игровая сцена

Прототип использует единый непрерывный top-down мир в одной Phaser-сцене. Интерьер дома собран из Basic Village и встроен в окружающую травяную область; игрок проходит через двухтайловый дверной проём без перехода сцен, камера следует за игроком в пределах мира, а весь world art и персонаж отрисовываются на единой 16×16 logical pixel grid при разрешении 320×180 с integer zoom.
