# Library: карта NestledBurrow

Этот файл — карта города. Он помогает найти нужную область проекта, не загружая в контекст всё подряд.

## Как пользоваться картой

- Основной чат сначала читает `PROJECT.md`, чтобы понять текущий вектор.
- Codex или субагент сначала читает эту карту, затем открывает только адреса, необходимые для задачи.
- Карта указывает маршрут, но не пересказывает содержимое документов.

## Действующие здания

### `PROJECT.md` — правительство

Текущее положение проекта, цель, следующий шаг, блокеры и решения, недавно изменившие направление.

Основной читатель: главный чат, управляющий работой.

Читать: перед планированием следующей задачи или после долгого перерыва.

## Технический район

### `src/`

Исходный код игры.

Читать: при реализации или проверке игрового поведения.

### `src/main.js`

Точка входа, текущая сцена минимального прототипа, создание комнаты, игрока и визуальных элементов интерфейса.

Читать: при изменении запуска, сцены, движения игрока или конфигурации Phaser.

### `src/input.js`

Каноническая логика ввода: параметры мобильного джойстика, dead zone, ограничение длины вектора и определение touch/coarse pointer устройств.

Читать: при изменении клавиатурного или мобильного управления.

### `scripts/check-input.mjs`

Автоматические проверки расчётов и ключевых сценариев ввода.

Читать: при изменении логики джойстика, объединения направлений или ограничений движения.

### `package.json`

Зависимости и команды запуска, сборки, предпросмотра и технических проверок.

Читать: при настройке окружения, зависимостей, сборки или команд проверки.

## Инфраструктурный район

### `.github/workflows/pr-check.yml`

Проверка pull request: установка зависимостей и production-сборка проекта.

Читать: только при изменении обязательных проверок PR.

### `.github/workflows/deploy-pages.yml`

Сборка и публикация GitHub Pages, генерация `/version.json`, проверка живой опубликованной версии и выставление статуса `pages/live`.

Читать: при изменении публикации, идентификатора сборки или правила «можно запускать».

## Архив

Старые handoff-файлы, Telegram-экспорт и прежние документы памяти не являются действующими адресами. Обращаться к ним нужно только для конкретного вопроса или восстановления утраченного обоснования.

## Правила развития города

- Новое здание появляется только после возникновения самостоятельной области знаний.
- Для здания заранее определяются назначение, читатель и момент чтения.
- Один факт имеет один канонический адрес; другие документы только ссылаются на него.
- Нельзя создавать пустые здания на будущее или дублировать сведения.
- При появлении, переименовании или удалении здания обновляется эта карта.
