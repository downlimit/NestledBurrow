# Library: карта NestledBurrow

Этот файл — карта города. Он помогает найти нужную область проекта, не загружая в контекст всё подряд.

## Как пользоваться картой

- Основной чат сначала читает `PROJECT.md`, чтобы понять текущий вектор.
- Codex или субагент сначала читает `AGENTS.md`, затем эту карту и только после этого открывает адреса, необходимые для задачи.
- Карта указывает маршрут, но не пересказывает содержимое документов.

## Действующие здания

### `PROJECT.md` — правительство

Текущее положение проекта, цель, следующий шаг, блокеры и решения, недавно изменившие направление.

Основной читатель: главный чат, управляющий работой.

Читать: перед планированием следующей задачи или после долгого перерыва.

## Технический район

### `AGENTS.md`

Обязательные правила работы Codex: подготовка, автоматические проверки, ручной runtime self-test, протокол подключения сторонних spritesheet и формат отчёта о проверке.

Читать: в начале каждой задачи Codex.

### `ASSETS.md`

Канонический адрес внешних ассетов, их источников, лицензий, выбранных файлов и итоговых путей в проекте.

Читать: при добавлении, удалении или проверке сторонней графики и лицензий.

### `src/`

Исходный код игры.

Читать: при реализации или проверке игрового поведения.

### `src/main.js`

Точка входа, текущая сцена минимального прототипа, загрузка semantic atlas комнаты и кадров персонажа, создание комнаты, анимаций и визуальных элементов интерфейса.

Читать: при изменении запуска, сцены, движения игрока, комнаты или конфигурации Phaser.

### `src/input.js`

Каноническая логика ввода: параметры мобильного джойстика, dead zone, ограничение длины вектора и определение touch/coarse pointer устройств.

Читать: при изменении клавиатурного или мобильного управления.

### `src/visualConfig.js`

Runtime-конфигурация масштаба, кадров персонажа и путей к проверенному semantic atlas комнаты.

Читать: при изменении персонажа, анимаций, масштаба или загрузки atlas.

### `src/kenneyRoomConfig.json`

Канонический машинный манифест комнаты: лицензия и происхождение Kenney pack, геометрия исходного spritesheet, исходные frame ID и SHA-256 выбранных тайлов, смысловые имена atlas, wall bands, atlas hash и утверждённый pixel hash предпросмотра.

Читать: при аудите, замене или повторном выборе тайлов комнаты. Raw frame ID не должны переноситься из этого файла в gameplay-код.

### `src/roomLayout.js`

Чистая логика сборки прямоугольной комнаты из смысловых имён `floor`, боковых стен и трёх проверенных wall bands.

Читать: при изменении формы комнаты, количества рядов стен или правил раскладки тайлов.

### `scripts/check-input.mjs`

Автоматические проверки расчётов и ключевых сценариев ввода.

Читать: при изменении логики джойстика, объединения направлений или ограничений движения.

### `scripts/check-visual.mjs`

Автоматические проверки файлов персонажа, направлений и idle, целостности semantic atlas, его SHA-256, источников выбранных Kenney-тайлов и структуры room layout.

Читать: при изменении визуальных ассетов, манифеста или конфигурации комнаты.

### `scripts/check-room-preview.py`

Pixel-exact генератор полного предпросмотра комнаты 960×540. Создаёт `artifacts/room-preview.png` и сравнивает его пиксели с утверждённым SHA-256 из `src/kenneyRoomConfig.json`.

Читать: при изменении room atlas, раскладки, масштаба или визуальной регрессии комнаты.

### `requirements-dev.txt`

Закреплённая Python-зависимость Pillow для воспроизводимой генерации предпросмотра комнаты в локальной среде и GitHub Actions.

Читать: при изменении Python-проверок визуального результата.

### `package.json`

Зависимости и команды запуска, сборки, предпросмотра и технических проверок. Команда `npm run check` объединяет input check, asset/layout check, pixel-exact room preview и production build.

Читать: при настройке окружения, зависимостей, сборки или команд проверки.

## Инфраструктурный район

### `.github/workflows/pr-check.yml`

Проверка pull request: установка Node/Python-зависимостей, все project checks, production-сборка и загрузка `room-preview-pr-<номер>` для обязательного визуального просмотра перед merge.

Читать: только при изменении обязательных проверок PR или workflow-артефакта визуального предпросмотра.

### `.github/workflows/deploy-pages.yml`

Сборка и публикация GitHub Pages, генерация `/version.json`, проверка живой опубликованной версии и выставление статуса `pages/live`.

Читать: при изменении публикации, идентификатора сборки или правила «можно запускать».

## Архив

Старые handoff-файлы, Telegram-экспорт и прежние документы памяти не являются действующими адресами. Обращаться к ним нужно только для конкретного вопроса или восстановления утраченного обоснования.

## Правила развития города

- Новое здание появляется только после возникновения самостоятельной области знаний.
- Для здания заранее определяются назначение, читатель и момент чтения.
- Один факт имеет один канонический адрес; другие документы только ссылаются на него.
- Нельзя создавать пустые здания на будущее или дублировать сведения.
- При появлении, переименовании или удалении здания обновляется эта карта.
