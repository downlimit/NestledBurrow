# Задача: fullscreen и динамический мобильный джойстик

## Цель

Улучшить управление экраном и мобильный ввод:

1. добавить кнопку переключения полноэкранного режима на desktop и смартфонах;
2. заменить фиксированный мобильный джойстик на динамический: первое касание в левой половине игрового экрана создаёт джойстик под пальцем, после отпускания он исчезает.

Текущий velocity-based movement, Basic Village, keyboard controls, collision, camera, build ID, debug-панель движения и pixel-grid поведение должны сохраниться.

## 1. Полноэкранный режим

### Поведение

Добавить компактную кнопку fullscreen в правом верхнем углу игрового контейнера.

- Кнопка видна на desktop и mobile, когда стандартный Fullscreen API доступен.
- Первое нажатие вызывает fullscreen для контейнера `#game`, а не только для canvas и не для случайного дочернего элемента.
- Повторное нажатие выходит из fullscreen.
- Выход через `Esc` или системный жест корректно обновляет состояние и иконку кнопки.
- Если Fullscreen API недоступен, кнопка не отображается; не делать фальшивый CSS-fullscreen.
- Ошибка или отклонение `requestFullscreen()` не должны ломать игру и создавать unhandled promise rejection.
- При `fullscreenchange` нужно сбросить активный joystick и повторно синхронизировать integer zoom/размер canvas.
- Не добавлять принудительную блокировку ориентации в рамках этой задачи.

### UI

- Кнопка должна находиться внутри `#game`, чтобы оставаться видимой в fullscreen.
- Использовать DOM-кнопку с корректным `type="button"`, `aria-label` и `title`.
- Иконка входа/выхода может быть выполнена inline SVG или CSS без новой зависимости и без внешнего ассета.
- Кнопка не должна перекрывать build ID, movement debug panel или игровое управление. Build ID сохранить, при необходимости аккуратно сместить.
- Нажатие кнопки не должно активировать динамический joystick.
- UI не должен менять логическое разрешение 320×180 или масштаб world art.

Желательно вынести работу с Fullscreen API в небольшой чистый модуль, например `src/fullscreen.js`, чтобы поведение можно было тестировать с mock document/element без Phaser.

## 2. Динамический джойстик

### Активация

Текущий постоянный joystick заменить следующим поведением:

1. Пока joystick не активен, его база и ручка скрыты.
2. Первое допустимое касание/нажатие pointer в левой половине игрового canvas активирует joystick.
3. Центр базы ставится под точку первого касания.
4. Если касание слишком близко к краю, центр ограничивается так, чтобы база полностью помещалась внутри canvas и не заходила в правую половину.
5. В момент активации movement vector равен нулю; движение начинается только после отклонения пальца от созданного центра.
6. База остаётся неподвижной до отпускания. Не реализовывать плавающую базу, которая догоняет палец.
7. После `pointerup`, `pointerupoutside`, `pointercancel`, `lostpointercapture`, `touchcancel`, `gameout`, blur, скрытия вкладки или `fullscreenchange` joystick полностью сбрасывается и скрывается.

### Ограничения

- Активировать joystick только на touch/coarse-pointer устройствах, как сейчас.
- Область активации — вся левая половина игрового canvas, а не фиксированный круг.
- Правая половина не должна активировать joystick: она остаётся свободной под будущие action buttons.
- Отслеживать конкретный активный `pointerId`; второй палец не должен перехватывать joystick.
- Касания DOM UI, fullscreen-кнопки и movement debug panel не должны создавать joystick.
- Сохранить текущие `baseRadius`, `knobRadius`, `maxOffset`, `deadZoneRatio`, аналоговую силу и визуальный стиль, если изменение не требуется для динамического позиционирования.
- Координаты и clamp рассчитывать в логических Phaser-координатах, а не в сырых CSS-пикселях окна.
- Keyboard input должен работать без изменений одновременно с новой joystick-логикой.

### Архитектура

Убрать зависимость расчёта joystick state от глобально фиксированных `centerX/centerY`.

Предпочтительная модель:

- `JOYSTICK` хранит размеры и коэффициенты;
- runtime state хранит текущий центр;
- чистая функция вычисляет vector/knob position относительно переданного центра;
- отдельная чистая функция проверяет левую половину и clamp допустимого центра.

Не помещать DOM fullscreen-логику внутрь `input.js`.

## Автоматические проверки

Расширить проверки и включить их в существующий `npm run check`.

Минимально проверить:

### Dynamic joystick

- касание в безопасной точке левой половины сохраняет центр непосредственно под пальцем;
- центр clamp-ится у левого, верхнего и нижнего края;
- центр не может выйти в правую половину;
- правая половина отклоняется как зона активации;
- pointer точно в центре даёт нулевой movement vector;
- dead zone сохраняется;
- максимальное отклонение даёт нормализованную аналоговую силу без превышения длины 1;
- разные runtime-центры дают корректные независимые результаты;
- существующие input и movement тесты остаются зелёными.

### Fullscreen

При вынесении API в модуль проверить mock-объектами:

- вход в fullscreen вызывает `requestFullscreen()` на `#game`;
- повторный toggle вызывает `document.exitFullscreen()`;
- unsupported state определяется корректно;
- rejected Promise обрабатывается без необработанного исключения;
- состояние определяется по `document.fullscreenElement`.

Не заменять полноценный browser self-test только regex-проверками исходного текста.

## Ручная проверка в браузере

Проверить до создания PR:

### Desktop

- кнопка видна и не перекрывает HUD;
- вход в fullscreen;
- выход повторным нажатием;
- выход через `Esc`;
- иконка/label соответствуют реальному состоянию;
- после переходов fullscreen сохраняются integer zoom, чёткие пиксели, camera follow и keyboard movement;
- обычный URL и `?movementDebug=1` работают, панели не перекрываются.

### Mobile/coarse pointer

- joystick изначально скрыт;
- появляется под пальцем в нескольких точках левой половины;
- корректно clamp-ится у всех краёв;
- база не двигается вслед за пальцем;
- отпускание скрывает joystick и персонаж плавно останавливается;
- резкое движение, диагонали и аналоговая сила ощущаются как до изменения;
- касание правой половины не создаёт joystick;
- второй палец не перехватывает активный joystick;
- fullscreen-кнопка не активирует joystick;
- вход и выход из fullscreen во время и после управления не оставляет залипшего movement vector;
- проверить portrait и landscape viewport без требования принудительно блокировать ориентацию.

Также повторно проверить doorway, wall sliding, world bounds и отсутствие diagonal speed boost.

- правый joystick;
- combat/aim input;
- orientation lock;
